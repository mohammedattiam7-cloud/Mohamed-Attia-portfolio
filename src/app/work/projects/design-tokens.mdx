## Overview

A scalable token system built to support multiple brands and contexts from a single source of truth. Instead of designing one theme and duplicating it, the system separates raw values, meaning, and component usage into three layers — so a new brand, language, or density swaps in by changing a mode, not by rebuilding screens.

## The Challenge

The seven theme dimensions don't add up — they multiply. A single screen could be Diamond × Dark × Arabic × Serif × Compact × Pills × Raised, which is one of 576 valid combinations.

The real problem was never visual styling. It was structure: how do you build a system where hundreds of theme combinations stay consistent, maintainable, and ready for engineers — without the file collapsing into chaos?

## Handoff to Code

A design system only matters if engineers can use it. Each theme axis exports as its own CSS file, so developers pull only the layers they need and compose the final runtime theme — brand + mode + language + density + radius + shadow.
